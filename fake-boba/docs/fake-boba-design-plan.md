# "假装喝奶茶" 第一版前端设计方案

## 摘要

为"假装喝奶茶"情绪产品设计第一版前端页面。这是一个纯虚拟的情绪玩具，帮助用户在"很想喝奶茶但暂时不喝"时快速获得心理满足感。第一版围绕核心闭环：**打开 → 选配方 → 模拟制作 → 假装喝几口 → 生成结果卡 → 分享**。

技术方案：**单文件 HTML/CSS/JS**，移动端优先，无框架依赖，所有视觉效果用 CSS/SVG 绘制。

## 二、5个核心场景设计

### 场景一：首页/入口

**目标**：2秒内让用户理解这是什么、产生好奇心、点击进入。

**布局**（从上到下）：
1. 氛围背景层：漂浮气泡粒子（CSS animation，8-12个随机大小/位置/速度的半透明圆）
2. 品牌标题区：大标题"假装喝奶茶" + 副标题"Fake Boba"
3. 情绪共鸣文案："很想喝但暂时不喝？来假装喝一杯吧！"
4. 两个CTA按钮：
   - 主按钮（渐变粉橙色）："🎲 随机来一杯" → 直接跳过选配方，随机选一个预设配方进入制作
   - 次按钮（描边样式）："📋 自己选配方" → 进入选配方页
5. 社交证明："已有 12,345 人假装喝过"

**入场动画时序**：背景渐变淡入(0ms) → 标题弹入(200ms) → 副标题(400ms) → 文案淡入(600ms) → 主CTA弹入(800ms) → 次CTA(1000ms) → 社交证明(1200ms)

### 场景二：选配方页

**目标**：选配方本身就是有趣的体验，不是枯燥表单。

**交互方式**：分4步横向滑动切换，每步一个类别。

| 步骤 | 类别 | 选项 | 交互 |
|------|------|------|------|
| 1 | 杯型 | 经典杯/胖胖杯/高瘦杯 | 图标卡片点选 |
| 2 | 茶底 | 红茶/绿茶/乌龙/芋泥/芒果/黑糖 | 网格卡片点选（含色块圆点） |
| 3 | 小料 | 珍珠/椰果/布丁/仙草/芋圆/红豆/奥利奥/奶盖 | 胶囊标签多选（最多3个） |
| 4 | 甜度+冰度 | 各4-5档 | 分段选择器 |

**动效**：步骤切换左右滑动(350ms)、选项点击弹性缩放(300ms)、顶部进度条过渡(300ms)。

### 场景三：制作页

**目标**：把"制作奶茶"变成有仪式感的小游戏。

**3个子阶段**：

| 阶段 | 动作 | 时长 | 交互 |
|------|------|------|------|
| 3a 倒茶底 | 茶壶倾斜 → 茶流倒入 → 液面上升 | 2s | 自动播放 |
| 3b 加小料 | 小料依次从上方掉落，沉底或浮起 | 每个料1.5s | 自动播放 |
| 3c 摇一摇 | 杯子左右摇晃，进度条增长 | 2-3s | DeviceMotion / 点击按钮 |

**关键动画**：
- 茶壶倾斜 `rotate(-30deg)` + 茶流 `height` 增长 + 液面 `height` 上升（ease-out）
- 液面顶部波浪：`border-radius` keyframes 循环变化
- 小料掉落：`translateY` + 弹性曲线，珍珠沉底、奶盖浮面
- 摇晃：杯子 `rotate + translateX` keyframes，液体延迟50ms惯性晃动，圆弧进度条（SVG stroke-dashoffset）

### 场景四：品尝页

**目标**：每喝一口都有满足感反馈。

**布局**：杯子居中（含吸管、蒸汽、液面、可见小料）+ "喝一口！"按钮 + 已喝进度(🧋🧋🧋⚪⚪) + 随机情绪文案

**每次点击"喝一口"的动画序列（1.2s）**：
1. 按钮按下 `scale(0.92)`
2. 吸管内液滴向上移动
3. 液面高度减少 ~20%（带波浪晃动）
4. 杯内小料随液面下降
5. 弹出情绪文案气泡（1.5s后淡出）
6. 进度圆点更新

**共5口**，喝完最后一口触发：杯子微倾 + 满足表情 + 纸屑星星粒子飞散。

**情绪文案池**："嗯~ 好满足！"/"珍珠QQ的！"/"奶味刚刚好~"/"热量？不存在的！"等10条随机。

### 场景五：结果卡/分享页

**目标**：生成好看的结果卡，用户愿意截图分享。

**卡片尺寸**：345×580px（适配朋友圈/小红书）

**卡片内容**：品牌标识 + "今天我假装喝了" + 杯子缩略图(CSS绘制) + 配方名称 + 甜度冰度 + 小料 + 个性化文案 + 热量节省数据 + 二维码占位区

**操作按钮**：保存图片 / 再来一杯 / 分享给朋友

**技术**：html2canvas 将DOM转Canvas导出PNG，优先 Web Share API（移动端原生分享），降级为下载图片。

## 三、关键交互动效技术方案

### 液体填充
- `height` transition 控制液面升降
- `border-radius` keyframes 模拟液面波浪
- `linear-gradient` 三色渐变模拟液体深浅
- `::before` 伪元素做波浪顶层

### 摇晃杯子
- `transform: rotate() translateX()` keyframes，10帧递减振幅
- 液体延迟50ms惯性晃动
- DeviceMotion API 检测摇晃（阈值15，需8次），降级为点击按钮

### 喝奶茶
- 液面 `height` 减少 + `border-radius` 波浪
- 吸管内液滴 `translateY` 向上
- 蒸汽加速（`animation-duration` 缩短）
- 纸屑 `translateY rotate` 飞散

### 蒸汽/气泡
- 蒸汽：3-5个白色模糊圆，`translateY scale opacity` keyframes
- 气泡：3-5个半透明圆，`translateY translateX` 带随机偏移

### 性能原则
- 所有动画只用 `transform` 和 `opacity`（GPU加速）
- `will-change: transform` / `contain: layout style paint`
- 动画元素不超过15个同时运行
- `prefers-reduced-motion` 媒体查询降级

## 五、技术实现要点

### 页面切换
全屏场景 `100vh`，`transform: translateX()` + `transition: 0.5s`，维护场景栈。

### 分享功能
html2canvas CDN 按需加载，`scale: 2` Retina清晰，`await document.fonts.ready` 等字体加载，截图前隐藏 `filter: blur()` 元素。

### 移动端适配
- `touch-action: manipulation` 禁用双击缩放
- `env(safe-area-inset-bottom)` 安全区域
- 杯子尺寸 `min(280px, 75vw)` 小屏适配
- 320px 最小宽度支持

### 文件结构
单文件 `index.html`：
- `<head>`：meta + Google Fonts + `<style>` + html2canvas(async)
- `<body>`：`#app` > 5个 `section.scene` + `<script>`

## 七、假设与决策

1. **第一版不做音效**：预留事件钩子，但MVP不实现
2. **杯型选择保留但非核心**：如果时间紧张可固定为经典杯
3. **二维码区域预留占位**：后续接入实际链接
4. **社交证明数字为假数据**：后续接入真实统计
5. **所有视觉用CSS/SVG绘制**：不使用图片资源，保证加载速度
6. **单文件交付**：所有代码在一个 index.html 中，方便预览和传播

## 八、验证方式

1. 移动端浏览器打开，检查5个场景流畅切换
2. 完整走一遍"随机来一杯"流程，确认所有动画正常
3. 完整走一遍"自己选配方"流程，确认4步选择正常
4. 测试"喝一口"5次，确认液面下降、文案弹出、喝完效果
5. 测试结果卡生成和保存图片
6. 在 320px/375px/428px 宽度下检查布局
7. 检查 `prefers-reduced-motion` 降级是否生效
