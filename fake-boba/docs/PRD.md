# 假装喝奶茶 - 产品需求文档（PRD）

> 版本：v2.0 | 更新：2026-04-23

## 一、产品概述

**产品名称**：假装喝奶茶（Fake Boba）
**产品定位**：一款趣味互动网页应用，让用户在没有真实奶茶的情况下，通过模拟"选配方→制作→品尝"的完整流程获得心理满足感
**目标用户**：想喝奶茶但暂时不方便/不想喝的年轻人（18-35岁）
**部署方式**：GitHub Pages 静态页面，移动端优先（375px 基准）
**技术栈**：Vue 3 CDN + 单文件 HTML/CSS/JS（无构建工具）

---

## 二、用户流程

```
首页 → 选配方 → 制作奶茶 → 品尝奶茶 → 结果卡
 │       │          │            │           │
 │    两条路径    雪克杯调制    喝5口      保存/分享
 │   ┌───────┐   摇晃+倒入    特效+情绪    再来一杯
 │   │随机   │                            回首页
 │   │自选   │
 │   └───────┘
```

### 2.1 首页（Home）
- **核心元素**：奶茶杯插画（带蒸汽）、标题"假装喝奶茶"、副标题"Fake Boba"
- **引导文案**："很想喝但暂时不喝？来假装喝一杯吧！"
- **两个入口**：
  - "🎲 随机来一杯"：从预设配方中随机选一个，直接进入制作
  - "📋 自己选配方"：进入4步选配方流程
- **社交证明**："已有 XX,XXX 人假装喝过"（数字滚动动画）
- **背景**：柔和渐变 + 浮动光斑

### 2.2 选配方（Recipe）— 4步流程

**第1步：选杯型**
- 3个选项：经典杯（标准）、胖胖杯（可爱）、高瘦杯（修长）
- 每个杯型用 CSS 绘制缩略图
- 选中后粉色边框高亮

**第2步：选茶底**
- 6个选项：红茶、绿茶、乌龙、芋泥、芒果、黑糖
- 每个选项显示对应颜色色块 + 名称
- 选中后边框高亮

**第3步：选小料（多选，最多3个）**
- 8个选项：珍珠、芋圆、椰果、红豆、仙草、布丁、芝士奶盖、奥利奥碎
- 胶囊按钮样式，选中后填充色
- 底部显示"已选 X/3 个小料"

**第4步：甜度 + 冰度**
- 甜度5档：无糖、三分甜、五分甜、七分甜、全糖
- 冰度5档：去冰、少冰、正常冰、多冰、热饮
- 分段控件样式

**导航**：
- 顶部进度条（25%→50%→75%→100%）
- "上一步"/"下一步"按钮（下一步在当前步骤未选择时 disabled）
- 左上角返回按钮（回首页）
- 最后一步"下一步"变为"开始制作"

### 2.3 制作奶茶（Making）— 两阶段

**阶段1：调制（雪克杯）**
```
流程：倒茶 → 加冰 → 加小料 → 摇晃
```
- 显示雪克杯（梯形透明杯 + 半圆盖）
- 倒茶：液体从上方流入雪克杯，液面上升（2秒）
- 加冰：冰块掉入杯中（数量根据冰度设置）
- 加小料：每个小料依次掉入，显示趣味文字（"珍珠Q弹登场！"）
- 摇晃：长按按钮摇晃，进度环从 0% 到 100%
  - 摇晃时雪克杯左右摆动，液体晃荡，气泡上升
  - 支持手机陀螺仪摇晃（降级为长按）

**阶段2：出品（倒入饮用杯）**
```
流程：雪克杯倾斜 → 液体流入饮用杯 → 饮用杯填满 → 跳转品尝
```
- 雪克杯向右倾斜 45°，液体流出
- 饮用杯出现在右侧，液面逐渐上升
- 如有芝士奶盖，最后铺在液面上
- 倒完后雪克杯淡出，饮用杯居中
- 自动跳转品尝页

### 2.4 品尝奶茶（Tasting）
- 显示饮用杯（带吸管），液面 80%
- 用户点击"喝一口"按钮，共 5 口
- 每喝一口：
  - 液面下降 16%
  - 吸管底部产生漩涡
  - 从吸管口飘出星星/爱心
  - 显示随机情绪气泡（"嗯~ 好满足！"）
  - 进度点更新（空心→实心）
- 第3口后：背景叠加暖色遮罩（满足感）
- 第5口（喝完）：
  - 杯子倾斜 15°
  - 弹出笑脸 + 彩色纸屑
  - 显示"查看结果卡"按钮

### 2.5 结果卡（Result）
- 显示本次配方信息：杯型、茶底、小料、甜度、冰度
- 个性化文案（根据配方特征生成）
- 心情标签（"快乐全糖党"、"佛系控糖派"等）
- CSS 绘制的小杯子缩略图
- 两个按钮：
  - "📸 保存图片"（html2canvas 截图保存）
  - "🔄 再来一杯"（重置状态，回首页）

## 四、技术架构

### 4.1 文件结构
```
fake-boba/
├── index.html          # 入口（Vue CDN + 内联CSS/JS）
├── build.sh            # 构建脚本（多文件→单文件，开发用）
├── css/style.css       # 全部样式
├── js/
│   ├── app.js          # Vue 应用入口
│   ├── data/constants.js
│   ├── store/state.js
│   ├── composables/    # useScene, useMaking, useTasting, useDeviceMotion
│   ├── components/     # CupComponent, EmotionBubble, ShakeProgress
│   └── scenes/         # Home, Recipe, Making, Tasting, Result
└── assets/favicon.png
```

### 4.2 关键约束
- Vue 3 CDN（`vue.global.prod.js`），不使用 ES Module
- 部署为单文件 HTML（CSS/JS 内联）
- 场景切换用 Vue `v-if` 条件渲染
- 所有动画只用 `transform` 和 `opacity`（GPU 加速）
- 移动端优先 375px
- 不使用外部 JS 库（html2canvas 除外）

### 4.3 已知 Bug
- **选配方点击无响应**：Vue 3 `data()` 返回 reactive 对象后，template 中响应式追踪不完整
- **修复方向**：用 `setup()` + `toRefs(state)` 解构，或将 state 属性直接作为组件 data

## 五、验收标准

1. 首页正常显示，两个入口按钮可点击
2. "自己选配方"：4步流程正常，选项可选中高亮，下一步按钮状态正确
3. "随机来一杯"：直接进入制作
4. 制作页：雪克杯调制→倒入饮用杯，动画流畅
5. 品尝页：5口喝完，特效正常（漩涡、星星、情绪气泡）
6. 结果卡：配方信息正确，保存图片功能正常
7. "再来一杯"：状态完全重置
8. 手机浏览器（含微信）完整流程无白屏无报错
